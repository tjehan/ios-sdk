SPTTrackPlayer Class Reference

Inherits from NSObject
Declared in SPTTrackPlayer.h

Overview

This class ties together SPTAudioStreamingController and SPTCoreAudioController to provide a basic audio player. For more advanced playback setups, use the aforementioned classes directly.

Attempts to enable playback with the given session.

- (void)enablePlaybackWithSession:(SPTSession *)session callback:(SPTErrorableOperationCallback)block

Parameters

The session to use for audio playback. Must be valid and authenticated with the SPTAuthStreamingScope scope.

The block to call when playback is enabled (or not). If playback could not be enabled, the block’s error parameter will contain information.

playTrackProvider:

Play the given track provider.

- (void)playTrackProvider:(id<SPTTrackProvider>)provider

Parameters

provider

The track provider to play.

Discussion

Playback of any current provider will be stopped and the given provider will be played from the beginning.

Equivalent to -[SPTTrackPlayer playTrackProvider:provider fromIndex:0];

skipToPreviousTrack:

Skips playback to the previous track.

- (void)skipToPreviousTrack:(BOOL)disallowSameTrackRestart

Parameters

disallowSameTrackRestart

If set to YES, playback will always be skipped to the track prior to the current one, bypassing default behaviour to restart the current track.

Discussion

The default behaviour of this method will restart the current track if playback is sufficiently progressed, matching the behaviour of most other media players.
